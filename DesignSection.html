The main interface element in my website is an a tag that is used to traverse between different pages. These tags look like buttons with a rounded border. These tags are extremely readible and contrast the background colours. I also have a search bar for if users are searching for something specific and they do not wish to navigate through menus. These are both navigational components. Lastly I am using a drop down menu as a way for employers to contact me personally or to find my social network accounts. This is a Input Control.

Implementation notes

a) My folder structure is laid out to sepearate the html and CSS documents and they are all in order of the hierarchy of the actual site. I used appropriate naming conventions for all styling and made sure to not use in-line styling anywhere throughout my site. The coding is modular in that it can be easily upgraded or changed completely.

b) I have tried my best in using semantic markup and have managed to clearly express my content through different containers, buttons, links and pages. Using Semantic markup makes my webiste optimised for an search engine. If someone were to search "Max Kruger Game Development", a search engline like google would be able to display my site as one of the answers. However, I could definately improve the accessibility of my website by creating more semantic tags to guide, for example, a blind person using an Ereader to navigate my website

d) Throughout my html pages I do not use a single in-line styling tag in order to create consistent styling throughout my site. The website itself is modular and thrghout my experience of creating it I have had to modify the pages countless times, proving that it is indeed modular and therefor maintainable.

Assignment 3 & Exam - Further Development

Explanation of Interaction Points

I have now managed to implement JavaScript into my website with the use of more buttons. My Blog section is no longer a tags but now uses buttons that use DOM Manipulation that replaces the window using the corresponding function. This has simplified my code and also helped with the layout of my Blogs site. The buttons have well used colours that contrast each other making it very readible and letting the user understand the difference between what is a heading and normal text. Furthermore I have used a golden border when the user hovers over the button which clearly indicates to the user what is selected and what they are about to click on. I have also changed my back a links to buttons that also ise DOM manipulation to return users to the blogs. The button also uses an golden border when hoeverd on. In my header I still use A tag with contrasting colours to highlight what is selected for the user for an easy understanding on what page they are on. Furthermore, I have completely transformed my headers navigation to use buttons with event listeners to include more javascript.

Further Style adaptations

I still use rounded borders which tie into my modern and minimilistic feel of the website however, using contrasting colours thoughout my website have lead to inconsistincy with headers. For my design section and weekly blogs I have removed the borders around my headers and just used and h1 and h2 headings that contrast with the background. I have decided to keep my font the same since Times new roman is very readible and comfortable on the eye.

Further Implementation notes

a) my folder structure has still been laid out separately and properly. My html and css documents are separate, as well as the javascript files. I have used appropriate naming conventions as to no confuse.

b) I have included more variety of semantic markup to further express my content and my intention behind the website. Later I do wish to include accessibility to E-readers who may be blind.

c) I have kept to html standards by not using in line syling but rather using css to do all of my styling on my different sites. I have updated my site to be far more consistent with regards to with my header and headings in different sections.

Previously my site was very badly tailored to responsive design but I have managed to change all layout CSS from fixed units like px to percentages and vh or vw which has enabled my site to react to different screen sizes and dimensions which is far better for a website.
